## 二、T09 — 渲染进阶技术

### 1. 一个片段着色器中的多个纹理
- 在单个 fragment shader 中绑定多个纹理单元 (`GL_TEXTURE0`, `GL_TEXTURE1`, ...)
- 使用 `uniform sampler2D` 分别采样不同纹理
- 通过纹理混合实现丰富表面效果（如：基础色图 + 法线贴图 + 镜面反射图）

### 2. 多个着色器对
- 不同对象使用不同的 shader program（例如：主物体用 `mainShader`，天空盒用 `skyboxShader`）
- 每个 shader program 包含自己的 vertex shader + fragment shader
- 切换使用 `glUseProgram(programID)`

### 3. 天空盒 (Skybox)
- 使用立方体贴图 (Cubemap) 渲染 6 个面的天空背景
- 渲染技巧：
  - **深度测试设置为 `GL_LEQUAL`**，确保天空盒永远在场景最远层
  - **移除视图矩阵的平移分量**（取视图矩阵的 3×3 部分），让天空盒跟随相机但不移动
  - 通常 **最后渲染** 或用深度技巧提前渲染
- 纹理加载：`glTexImage2D()` 分别加载 6 个面的图像

### 4. 法线贴图 (Normal Mapping)
- 用纹理存储法线向量，替代逐顶点法线，增加表面细节而无需增加几何复杂度
- 法线贴图的向量存储在切线空间 (Tangent Space)
- 需要计算 **切线 (Tangent)** 和 **副切线 (Bitangent)**，构建 TBN 矩阵
- TBN 矩阵用于将法线从切线空间变换到世界空间（或视图空间）

### 5. 实例化渲染 (Instanced Rendering)
- 一次性绘制多个相同几何体的副本，减少 draw call 次数
- 使用 `glDrawArraysInstanced()` / `glDrawElementsInstanced()`
- 每个实例的专属数据（如位置、颜色、缩放）通过 `glVertexAttribDivisor()` 传递

## 三、T10 — 动画与运动

### 1. 定时器驱动动画
- 使用 `glutGet(GLUT_ELAPSED_TIME)` 或 `glfwGetTime()` 获取经过时间
- 公式：`timer = (float)glfwGetTime()`
- 将 timer 用于对象的**平移矩阵**实现移动

### 2. 车辆移动
- 用 timer 控制车辆位置沿路径变化（如沿 Z 轴平移）
- 可以通过三角函数实现来回往返运动

### 3. 岩石旋转
- 用 timer 定义旋转角度：`angle = timer * speed`
- 将角度传入旋转矩阵：`glm::rotate(model, angle, axis)`

### 4. 关于色差
- 与演示视频可能存在颜色差异，这是库版本或系统差异导致的
- **关注功能实现而非精确颜色匹配**

## 四、T11中 — 交互式 3D 场景

### 1. 场景布局推荐
- 将所有模型放在 **XY 平面** 上
- 行星 (Planet) 和外星车辆 (Alien) 沿 **Z 轴** 放置
- 飞船 (Spacecraft) 在 XY 平面上自由移动

### 2. 三类坐标系
1. **航天器局部坐标系** — 物体自身的坐标系
2. **世界坐标系** — 全局场景坐标系
3. **原点在行星中心** — 场景的中心参考点

### 3. 模型矩阵计算
```
Model = Scale × Rotate × Translate
```
- 模型通常定义得很大 → 先用缩放矩阵缩小
- 旋转 + 平移实现运动
- 输入来自鼠标和键盘

### 4. 键盘控制旋转
- 定义按键回调函数获取键盘事件
- 根据按键更新物体的旋转角度
- 实现前进方向与朝向一致（即物体朝其正面方向移动）

### 5. 鼠标控制旋转
- 定义鼠标回调函数（鼠标移动回调）
- 水平移动 → 偏航 (Yaw)，垂直移动 → 俯仰 (Pitch)
- 示例：向左移动鼠标 → 航天器头部向左转

### 6. 视点设定
- **将虚拟相机固定在航天器尾部后方**，与航天器相对静止
- 观察方向与航天器头部指向一致（第三人称跟随视角）
- 视图矩阵计算：`glm::lookAt(cameraPos, cameraPos + cameraFront, cameraUp)`
- 透视矩阵设置：`glm::perspective(fov, aspect, near, far)`

## 五、T12中 — 碰撞检测与项目综合

### 1. 碰撞检测（基于距离）
- 最简单的碰撞检测：**使用距离**
- 公式：
  ```cpp
  vec4 vectorA = matrixA * vec4(0, 0, 0, 1);  // 物体A的世界坐标
  vec4 vectorB = matrixB * vec4(0, 0, 0, 1);  // 物体B的世界坐标
  
  if (distance(vectorA, vectorB) <= threshold)
      return true;  // 碰撞发生
  ```
- 碰撞发生后：改变对象外观（纹理变化、颜色变化等）
- 距离度量：欧氏距离

### 2. 碰撞后的反馈
- **航天器靠近本地太空飞行器** → 触发警报 → 改变外星车辆纹理（变色）
- **收集金牌** → 靠近黄金 → 金牌消失/变为岩石
- **全部收集完成** → 航天器纹理变为金色

### 3. 纹理切换机制
- 为对象准备多个纹理：
  - 飞船：Texture1（正常）、Texture2（金色）
  - 车辆：Texture1（正常）、Texture2（警报后变色）
- 通过碰撞检测触发纹理切换：`glBindTexture()` 重新绑定或切换 uniform

### 4. 加载 OBJ 文件的注意事项
- 使用 **Assimp** 库加载 `.obj` 文件
- 加载时使用 `aiProcess_Triangulate`（三角化）
- 需要计算每个顶点的 **切线 (Tangent)** 和 **副切线 (Bitangent)**（用于法线贴图）
- 生成 VAO/VBO 时包含切线和副切线数据：
  ```cpp
  glBufferData(GL_ARRAY_BUFFER, obj.tangents.size() * sizeof(glm::vec3), &obj.tangents[0], GL_STATIC_DRAW);
  glBufferData(GL_ARRAY_BUFFER, obj.bitangents.size() * sizeof(glm::vec3), &obj.bitangents[0], GL_STATIC_DRAW);
  ```
- **透明度是可选功能**，不强制实现

- [LearnOpenGL CN](https://learnopengl.com/) — 非常详细的 OpenGL 教程，含大多数技术的源代码
